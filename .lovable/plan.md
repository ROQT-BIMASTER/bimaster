

# Análise de Originalidade — Módulo de Projetos vs Asana

## Elementos Comuns com Asana (padrões de mercado, não exclusivos)

Estes são **padrões universais de project management** usados por dezenas de ferramentas (Monday, ClickUp, Linear, Notion, Jira). Não configuram plágio por si só:

- Visões Lista, Quadro (Kanban), Cronograma (Gantt), Calendário — presentes em **todas** as ferramentas
- Seções colapsáveis com contadores
- Checkbox de conclusão (círculo → check verde)
- Tabs horizontais com ícones
- Painel lateral (Sheet) para detalhe da tarefa
- Inline editing (duplo clique no título)
- "Adicionar tarefa" inline com ícone +

## Pontos que Mais se Assemelham ao Asana Especificamente

| Elemento | Similaridade | Risco |
|----------|-------------|-------|
| Grid de 11 colunas na lista (expand, checkbox, nome, produto, responsável, data, colab, criador, data mod, status, estágio) | Layout tabular denso = muito parecido com Asana List View | Médio |
| Tabs underline-style no header (Lista, Quadro, Cronograma, Calendário) com borda inferior ativa | Idêntico ao padrão visual do Asana | Médio |
| Botão "+" Adicionar tarefa no canto superior direito da toolbar | Posicionamento igual ao Asana | Baixo |
| Health pills horizontais (Concluídas, No prazo, Em risco, Atrasadas) | Similar ao "Project Status" do Asana | Baixo |
| Kanban cards com foto 16:9, checkbox, avatar stack no rodapé | Composição visual similar | Baixo |

## Elementos Já Originais (diferenciais que vocês já têm)

- **Briefing por seção** com importação IA de planilhas — Asana não tem
- **Cofre Oficial** com versionamento e aprovação documental — exclusivo
- **Produto vinculado** com thumbnail na lista e grade de display — específico do negócio
- **Ghost Trails** para rastreio de movimentação entre seções — não existe no Asana
- **Estágio de desenvolvimento** (Briefing → Lançamento) como coluna separada — workflow próprio
- **Modo Foco** com 3 colunas (detalhe + produto + chat) — layout exclusivo
- **Cor de fundo personalizável** por projeto — Asana não oferece
- **Resumo IA** do projeto — diferencial
- **Retrabalho** como tipo de tarefa com badge — específico

## Plano de Diferenciação Visual (sem quebrar funcionalidade)

### 1. Reformular o Header de Navegação
**Atual**: Tabs horizontais com underline (= Asana)
**Proposta**: Mudar para **pills/segmented control** com fundo arredondado no item ativo, em vez de borda inferior. Visualmente distinto, mesma funcionalidade.

### 2. Redesenhar a Barra de Saúde (Health Panel)
**Atual**: Pills horizontais inline
**Proposta**: Transformar em uma **barra de progresso segmentada** (stacked bar) colorida mostrando proporção visual de cada status. Mais compacto e visualmente único.

### 3. Ajustar Layout da Lista
**Atual**: Grid uniforme de 11 colunas (= Asana density)
**Proposta**: Agrupar visualmente as colunas em **blocos semânticos** com separadores sutis — (Identidade: expand+check+nome+produto) | (Pessoas: responsável+colab+criador) | (Tempo: data+modificação) | (Status: status+estágio). Adicionar linhas zebradas opcionais.

### 4. Kanban Cards — Adicionar Identidade Visual
**Proposta**: Adicionar uma **barra lateral colorida** (accent bar) no card baseada no estágio, e reorganizar o footer do card com layout diferente do Asana (ex: data à direita, avatares à esquerda — inverso do Asana).

### 5. Seções — Diferenciar Visualmente
**Atual**: Chevron + nome + contador (= Asana)
**Proposta**: Adicionar uma **barra de progresso mini** inline na seção (ao lado do contador), e usar um ícone de pasta/grupo em vez do chevron puro.

### 6. Detalhe da Tarefa (Sheet)
**Proposta**: Manter Sheet lateral mas adicionar o **header com gradiente** baseado no estágio da tarefa, diferenciando visualmente do painel branco flat do Asana.

---

## Resumo de Mudanças

- **6 ajustes visuais** que diferenciam sem alterar funcionalidade
- Foco em: navegação (pills vs underline), health (barra vs pills), lista (blocos vs grid uniforme), kanban (accent bar), seções (progress bar), detalhe (header gradiente)
- Nenhuma feature removida, apenas restyling de componentes existentes
- Mantém todos os diferenciais já construídos (Briefing IA, Cofre, Ghost Trails, etc.)
