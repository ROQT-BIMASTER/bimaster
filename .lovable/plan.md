# Configuração de Prazos, Painel de Metas IA e Relatórios Avançados\n\n## Objetivo\n\nPermitir que **Gerentes/Coordenadores** configurem prazos e metas no momento da criação do projeto, com cálculo correto de **dias úteis** (ignorando finais de semana e feriados nacionais via BrasilAPI), e disponibilizar um **Painel de Acompanhamento de Metas** com IA orientativa para planos de ação e um pacote de **Relatórios Avançados** (operacionais + executivos).\n\n---\n\n## 1. Banco de Dados — Novas Estruturas\n\n### 1.1 Configuração de calendário/prazos por projeto\nEstender `projetos` com:\n- `regime_calendario text default 'dias_uteis'` — valores: `corridos`, `dias_uteis`, `uteis_com_sabado`\n- `usa_feriados boolean default true`\n- `uf_feriados text default 'BR'` — para feriados nacionais ou estaduais\n- `data_inicio date` — início planejado\n- `data_fim_alvo date` — entrega-alvo (meta global)\n- `prazo_padrao_tarefa integer default 5` — dias padrão ao criar tarefa\n- `alerta_antecipacao_dias integer default 2`\n\n### 1.2 Tabela de feriados (`feriados`)\n- `id uuid pk`\n- `data date not null`\n- `nome text`\n- `tipo text` — `nacional`, `estadual`, `municipal`, `empresa`\n- `uf text` — null para nacional\n- `fonte text` — `brasilapi`, `manual`\n- `ano integer`\n- Índice único `(data, uf, tipo)`\n- RLS: leitura para autenticados; escrita só para Admin\n\n### 1.3 Metas formais do projeto (`projeto_metas`)\nDistinta das `projeto_tarefa_metas` (checklist de tarefa). Esta é macro-meta do projeto:\n- `id`, `projeto_id`, `titulo`, `descricao`\n- `tipo text` — `entrega`, `qualidade`, `prazo`, `custo`, `volume`\n- `valor_alvo numeric`, `valor_atual numeric default 0`, `unidade text`\n- `data_inicio date`, `data_alvo date`\n- `status text` — `em_andamento`, `concluida`, `atrasada`, `em_risco`\n- `peso integer default 1` — para ponderar score do projeto\n- `created_by`, `created_at`, `updated_at`\n- RLS: membros do projeto leem; Coordenador/Gerente/Criador escrevem\n\n### 1.4 Relação meta ↔ plano de ação\nAdicionar `meta_id uuid references projeto_metas(id)` em `projeto_planos_acao`.\n\n### 1.5 Cache de relatórios IA (`projeto_relatorios_ia`)\n- `id`, `projeto_id`, `tipo` (`status_semanal`, `burndown`, `analise_atrasos`, `produtividade`, `executivo`, `preditivo`)\n- `periodo_inicio`, `periodo_fim`\n- `conteudo_md text`, `dados_json jsonb`\n- `gerado_por`, `gerado_em`\n- Permite re-leitura sem recomputar (TTL 24h)\n\n### 1.6 Função SQL `calcular_data_util(data_base date, dias int, projeto_id uuid)`\nCalcula data final somando `dias` úteis a `data_base`, considerando regime do projeto e feriados em `feriados`. Usada por triggers em `projeto_tarefas` quando `data_inicio` é informado e prazo é em dias úteis.\n\n---\n\n## 2.
